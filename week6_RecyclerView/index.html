    任务6：RecyclerView

    作业目标

    掌握使用RecyclerView进行内容的动态展示。

    案例分析

    案例程序下载

    功能介绍

    点击单词后出现clicked！+相应的单词

    点击悬浮按钮（FAB）后在列表末尾按顺序出现+单词。

    代码讲解

    创建recycler view，创建adapter并提供展示数据，连接adapter与recycler view，将recycler view设置为默认线性布局。

    mRecyclerView = findViewById(R.id.recyclerview);
mAdapter = new WordListAdapter(this, mWordList);
mRecyclerView.setAdapter(mAdapter);
mRecyclerView.setLayoutManager(new LinearLayoutManager(this));

    在wordlist里面添加新单词，提醒adapter数据已经改变并将视角滚动到最底层。

    public void onClick(View view) {
    int wordListSize = mWordList.size();
    mWordList.addLast("+ Word " + wordListSize);
    mRecyclerView.getAdapter().notifyItemInserted(wordListSize);
    mRecyclerView.smoothScrollToPosition(wordListSize);
}

    获取点击位置，确定点击的单词，修改单词内容并通知adapter状态改变使得RecyclerView能够及时展示数据。

    int mPosition = getLayoutPosition();
String element = mWordList.get(mPosition);
mWordList.set(mPosition, "Clicked! " + element);
mAdapter.notifyDataSetChanged();

    创建一个新的自定义视图holder来展示RecyclerView。 itemView是显示数据的视图。 Adapter用来管理数据和视图

    public WordViewHolder(View itemView, WordListAdapter adapter) {
    super(itemView);
    wordItemView = itemView.findViewById(R.id.word);
    this.mAdapter = adapter;
    itemView.setOnClickListener(this);
}

    手动创建新的View或从XML布局文件中对其进行inflate 。 新的ViewHolder将通过onBindViewHolder显示adapter的元素。 缓存对View的子视图的引用，避免不必要的调用。

    public WordListAdapter.WordViewHolder onCreateViewHolder
    (ViewGroup parent,int viewType) {
        View mItemView = mInflater.inflate(
                R.layout.wordlist_item, parent, false);
        return new WordViewHolder(mItemView, this);
    }

    由RecyclerView调用以在指定位置显示数据。 此方法更新ViewHolder.itemView的内容，在给定位置设置内容。

    @Override
public void onBindViewHolder
(WordListAdapter.WordViewHolder holder,int position) {  
    String mCurrent = mWordList.get(position);
    holder.wordItemView.setText(mCurrent);
}

    作业说明

    作业要求

    本次作业的主要任务是通过RecyclerView在动态页面滚动显示多条动态，并实现动态发布功能（动态只要求实现文字版）。

    本次作业需要实现以下功能： 1. 通过RecyclerView实现在动态页面实现动态的滚动展示。 2. 要求在动态页面中一次展示10条动态，若有更多的动态，要求用户点击“查看更多”按钮，再加载10条动态，直到全部展示完毕。 3. 创建动态发布页面，实现发布新动态的功能。新的动态发布后，会在动态页面的顶部展示最新的动态。
